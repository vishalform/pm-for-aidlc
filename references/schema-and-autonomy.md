# Schema & Autonomy — the 6-axis read and the autonomy-gate decision table

The authoritative spec for how `pm-for-aidlc` reads a task and decides how to run
it. `scripts/autonomy_gate.py` implements this table exactly — when in doubt, run
the script; this doc explains *why* it returns what it returns.

## 1. The schema (6 axes + actuation)

Every AIDLC task is read with six axes: `leverage · blast · eval · regr · safety · fly`,
plus an **`actuation`** distinction that separates a passive monitor from an actuator.

| Axis | Source | Values | What it gates |
|---|---|---|---|
| **leverage** | `ai_leverage` | `autonomous` / `copilot` / `human-led` | Base execution mode |
| **blast** | `data/aidlc_axes.csv` | `low` / `med` / `high` / `unknown` | Forces human confirm at `high`; `unknown` blocks unattended |
| **eval** | `eval_coverage` | `none` / `partial` / `gated` | **The enabler of unattended running** |
| **regr** | `regression_risk` | `low` / `med` / `high` | Forces an eval-gate (+ canary at blast≥med) |
| **safety** | `safety_exposure` | `none` / `med` / `high` | `safety_gate` at `med`+; **human confirm at `high` actuating** (carve-out: read-only gated monitor) |
| **fly** | `flywheel_linkage` | `none` / `consumer` / `producer` | Producers carry compounding risk |
| **actuation** | bundle or `--actuation` | `read-only` / `reversible` / `irreversible` | Whether a safety:high task may run unattended |

`actuation` defaults to **`reversible` (actuating)**. The read-only monitor carve-out must be
**ASSERTED** — via `--actuation read-only` or the bundled catalog (`data/aidlc_axes.csv`, only
AT-56 today) — and is **never inferred** from low blast. (Inferring read-only from
`autonomous + blast=low` was the C-A safety hole: an actuating safety:high task at low blast
wrongly ran unattended.) Pass `--actuation` to override the bundle.

Core principle (the epistemic inversion): **`blast_radius` is necessary but
insufficient.** The dominant AI risk is **undetectable wrongness**, which `eval`,
`regr`, `safety`, and `fly` capture and `blast` does not. And a pre-release
`safety_gate` is not runtime oversight: a safety:high **actuator** keeps a human in the
loop — *the decision stays human* — regardless of how reversible the action is.

## 2. Where each axis comes from

- `leverage`, `eval`, `regr`, `safety`, `fly` are columns in `aidlc_microtasks.csv`.
- **`blast` is bundled** in `data/aidlc_axes.csv` (generated by `scripts/build_axes.py`,
  which parses it from `task-catalog.md` and **cross-checks the 5 CSV axes, failing on any
  drift**). So `autonomy_gate.py --id AT-56` resolves a precise blast (`low`) with no flag.
- **blast resolution order:** `--blast` flag → bundled catalog → **`unknown`**. An
  `unknown` blast is *not* a safe default — it **never authorizes an unattended run** (pass
  `--blast` to authorize one). There is no silent `med` default any more.
- **`actuation`** comes from the bundle (`read-only` for AT-56; `reversible` otherwise) or
  `--actuation`; when omitted it derives to **`reversible`** — the read-only carve-out is
  asserted, never inferred (see §1). Only AT-56 is `autonomous + safety:high`, so it is the
  only id whose read-only flag changes a verdict.
- CSV `–`/blank normalizes to `none`; `human` → `human-led`.

## 3. The three execution modes

| Mode | Meaning | Human in the per-run loop? |
|---|---|---|
| `run_unattended` | AI runs the task autonomously | No (but controls still apply) |
| `draft_then_confirm` | AI produces the artifact; a human reviews before it takes effect | Yes — passive review |
| `human_led` | AI preps context only; a human performs the task | Yes — human does it |

## 4. The controls (safety envelope)

| Control | Requirement |
|---|---|
| `eval_gate` | Output must pass the eval suite / CI gate before it counts. |
| `build_eval_first` | `regr=high` but `eval=none`: there is no gate to pass — **build the grader first**. |
| `canary_or_shadow` | Shipping-scale change (blast≥med): shadow → canary + online scoring + rollback. |
| `human_confirm` | Explicit human sign-off (stronger than passive review). |
| `provenance_log` | Log provenance because the output feeds the data flywheel (compounding). |
| `safety_gate` | Must pass the pre-release safety suite (jailbreak / abuse / PII). |
| `governance:runlayer_managed_only` | MCP selection (AT-26): governed servers only; flag shadow servers. |

## 5. The decision precedence (what the script does)

Applied in order; the **most restrictive** outcome wins. Exact logic in `autonomy_gate.py::verdict`.

1. **Base mode from `leverage`.** `human-led → human_led`; `autonomous → run_unattended` (tentative); `copilot → draft_then_confirm`.
2. **The key inversion.** Tentatively `run_unattended` **and** `eval ∈ {none, partial}` → `draft_then_confirm`. If `eval=gated`, add `eval_gate`.
3. **Regression risk.** If `regr=high`: add `eval_gate` (or `build_eval_first` when `eval=none`); add `canary_or_shadow` if `blast ∈ {med, high, unknown}`.
4. **Compounding risk.** `safety=med` adds `safety_gate` (run the safety suite) with **no**
   human downgrade. `safety=high` or `fly=producer`: add `provenance_log` (producer) and
   `safety_gate` (safety), then **downgrade to `draft_then_confirm` + `human_confirm`** when either:
   - `safety=high` **and not** a read-only monitor with a working gate (`is_read_only ∧ eval=gated`), **or**
   - `fly=producer` **and** `eval≠gated`.
   The **only** safety:high carve-out that stays unattended is a read-only, gated **monitor** (AT-56),
   and `is_read_only` is true only when asserted (bundle/`--actuation`), never inferred. Any actuator
   (reversible/irreversible) keeps a human in the loop.
5. **Operational irreversibility.** If `blast=high`: downgrade `run_unattended → draft_then_confirm` and add `human_confirm`.
6. **Unknown blast.** If `blast=unknown` and still `run_unattended`: downgrade + `human_confirm` ("pass `--blast` to authorize").
7. **Final guard.** `run_unattended` requires `eval=gated`; else downgrade.
8. **Trust reminder.** A `run_unattended` verdict notes that `eval=gated` must be a *judge-validated* grader (`judge_validation.py → TRUSTED`) and the axes honest, not self-reported.

## 6. Decision table (quick reference)

| Condition | Result |
|---|---|
| `leverage=human-led` | `human_led` (+ controls) |
| `autonomous + eval=gated + regr≤med + safety≠high + blast=low + fly≠producer` | **`run_unattended`** + `eval_gate` |
| `autonomous + eval∈{none,partial}` | `draft_then_confirm` (the inversion) |
| `regr=high + eval=none` | `build_eval_first` (not a phantom `eval_gate`) |
| `regr=high` (eval≥partial) | + `eval_gate` (+ `canary_or_shadow` if blast≥med) |
| `safety=med` | + `safety_gate` (run the safety suite; **no** human downgrade) |
| `safety=high` **actuating** (reversible/irreversible) | `draft_then_confirm`/`human_led` + `human_confirm` + `safety_gate` |
| `safety=high` **read-only monitor** + `eval=gated` (asserted) | keep mode + `safety_gate` (e.g. AT-56) |
| `fly=producer` + `eval≠gated` | force `human_confirm` + `provenance_log` |
| `fly=producer` + `eval=gated` | keep mode + `provenance_log` (e.g. AT-47) |
| `blast=high` | force `human_confirm` |
| `blast=unknown` | never `run_unattended` (pass `--blast`) |

## 7. Worked examples

All verified against `autonomy_gate.py` reading the bundled `data/aidlc_axes.csv`.

| Task | Axes (leverage·blast·eval·regr·safety·fly · actuation) | Verdict | Controls |
|---|---|---|---|
| **AT-56** Groundedness monitor | autonomous·low·gated·high·high·consumer · read-only | `run_unattended` | eval_gate, safety_gate |
| **AT-47** Contamination check | autonomous·med·gated·high·none·producer · reversible | `run_unattended` | canary_or_shadow, eval_gate, provenance_log |
| **AT-20** Prompt regression CI | autonomous·low·gated·high·med·none · reversible | `run_unattended` | eval_gate, safety_gate |
| **AT-08** Write eval rubric | human-led·med·gated·high·med·producer · reversible | `human_led` | canary_or_shadow, eval_gate, provenance_log, safety_gate |
| **AT-03** Thumbs-down triage | copilot·low·partial·low·med·producer · reversible | `draft_then_confirm` | human_confirm, provenance_log, safety_gate |
| **AT-53** Model rollback | human-led·high·gated·high·high·none · reversible | `human_led` | canary_or_shadow, eval_gate, human_confirm, safety_gate |
| **AT-72** Data-rights posture | human-led·high·none·low·high·producer · reversible | `human_led` | human_confirm, provenance_log, safety_gate |
| **C-A guard** actuating safety:high, blast:low, NO `--actuation` | autonomous·low·gated·low·high·none · reversible (derived) | `draft_then_confirm` | eval_gate, human_confirm, safety_gate |

The instructive contrasts: **AT-56 stays unattended** only because it is asserted a *read-only*
gated monitor in the bundle; the **C-A guard** row shares almost the same axes (and the same low
blast) but, with no `--actuation` it derives to *reversible* — an actuator — and drops to
`draft_then_confirm` + `human_confirm`. That derivation is the C-A fix: read-only is never assumed
for a safety surface. **AT-47** (gated producer) stays unattended with provenance; an *ungated*
producer would gain `human_confirm`. Note `safety=med` (AT-20/08/03) now carries a `safety_gate`.

## 8. The modality × risk-tier overlay (executable)

The autonomy verdict sets *whether* a human is in the loop; the overlay sets *how hard the
gates are*. It is **executable**: pass `--modality` and `--risk-tier` and the gate
recomputes on tightened axes (it only ever tightens, never loosens).

- **Modality:** `single-turn-rag` · `tool-using` · `coding` · `multi-agent` · `voice` · `image-gen` · `extraction`.
- **Risk-tier:** `internal` · `external-low-stakes` · `external-high-stakes`.

| Overlay signal | Effect (applied by the script) |
|---|---|
| risk-tier `external-high-stakes` | bump `safety` + `blast` one notch; add `canary_or_shadow` + `safety_gate`; set `gate_metric = pass^k`. |
| modality `tool-using/coding/multi-agent` | note: add tool-call-accuracy + transition-failure evals; bound steps/tokens (AT-27/28). |
| modality `voice` | note: add a latency-budget gate (p50/p95). |
| modality `single-turn-rag` | note: split retrieval recall@k vs groundedness (AT-23). |

Use `pass@k` in development / human-in-loop; **gate autonomous high-stakes deployment on
`pass^k`** (`eval_stats.py passk`; commonly cited target ≈ 0.95 — verify current).

## 9. Edge cases and how the gate resolves them

- **`blast` unknown** (no flag, not in the bundle): the gate **refuses to run unattended** and asks for `--blast`. This replaced the old (unsafe) silent `med` default.
- **Autonomous + safety=high + actuating**: forced to `draft_then_confirm` + `human_confirm` + `safety_gate`. Only a read-only gated monitor (AT-56) stays unattended, and **read-only must be asserted** (bundle/`--actuation`) — when `--actuation` is omitted it derives to `reversible`, so a low-blast safety actuator is correctly held for a human (the C-A fix; inferring read-only from low blast was the bug).
- **`safety=med`**: adds a `safety_gate` (run the pre-release safety suite) but does **not** force a human — the axis is 3-valued, not binary.
- **Producer + eval≠gated** (e.g. AT-33 preference-data, AT-09 golden dataset): `human_confirm` + `provenance_log` — a bad artifact compounds and nothing catches it.
- **`regr=high` + `eval=none`**: `build_eval_first` (you cannot "pass" a gate that does not exist), not a phantom `eval_gate`.
- **`phase_status=later`** (training cluster): the gate appends a "CONFIRM you are actually training" reason before you execute.
- **AT-26 (MCP selection)**: adds `governance:runlayer_managed_only` and a human-owned-approval reason.
- **Self-reported axes**: the gate is only as honest as the axes handed to it. A `run_unattended` verdict carries a reminder to confirm `eval=gated` is judge-validated and the axes are not an optimistic/injected self-report.
